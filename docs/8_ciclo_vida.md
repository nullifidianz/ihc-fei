1. **Características da Plataforma**

| Característica                                      | Descrição                                                                                                                                                                                                                                           |
| :-------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Descrição do Software                               | Aplicação Web cujo frontend será construído em React, backend em Java, utilizando Banco de Dados Relacional e infraestrutura Blockchain/Hyperledger (com smart contracts em Solidity)                                                               |
| Descrição do Hardware                               | Sistema multiplataforma. O acesso será feito primariamente via Desktops/Notebooks pelas Universidades e Recrutadores (uso em escritório), e via Smartphones e Notebooks pelos Estudantes (uso móvel e pessoal)                                      |

|                                                     | 3. |                                                     | 2. Dependência de Conexão Constante: Por ser um sistema Web/Blockchain, os usuários precisam de internet para qualquer operação (visualizar, emitir ou validar), o que pode gerar telas de erro se a conexão falhar                                 |
|                                                     | 3. Rigidez na Importação de Dados: O sistema depende de arquivos CSV bem formatados ou integração (que hoje não existe) com sistemas acadêmicos legados (como o SIGA), exigindo tratamento prévio de erros de formatação                            |

3. **Metas de Usabilidade**
   1. **Qualitativo**
      Prevenção e Recuperação de Erros: O sistema deve validar CPFs automaticamente, alertar sobre inconsistências de dados antes da emissão e salvar rascunhos para evitar perda de dados por expiração de sessão
      .
      Visibilidade do Status do Sistema: Implementar barras de progresso claras durante a emissão em lote e relatórios detalhados indicando exatamente quais diplomas falharam e o motivo
      .
      Acessibilidade e Autoatendimento: Estudantes e empresas devem conseguir visualizar, baixar (PDF) e validar (QR Code/Hash) os diplomas sem necessidade de acionar o suporte ou a secretaria
      .
   2. **Quantitativo**
